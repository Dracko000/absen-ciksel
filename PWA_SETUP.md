# Progressive Web App (PWA) Setup

## Features

This application is configured as a Progressive Web App with the following features:

- **Installable**: Users can install the app on their devices like a native application
- **Offline Support**: Basic functionality available when offline
- **Responsive**: Works on all device sizes
- **Fast Loading**: Optimized performance with caching
- **Secure**: Served over HTTPS

## Configuration

The PWA is configured in:
- `next.config.js` - Configured with next-pwa
- `public/manifest.json` - Web app manifest
- `public/icon-*.png` - App icons

## Icons

For a complete PWA experience, you'll need to replace the placeholder icon files with actual app icons:

1. `public/icon-192x192.png` (192x192 pixels)
2. `public/icon-256x256.png` (256x256 pixels) 
3. `public/icon-384x384.png` (384x384 pixels)
4. `public/icon-512x512.png` (512x512 pixels)

## Development vs Production

- PWA features are disabled in development mode
- PWA features are enabled in production builds

To test PWA features locally, build and serve the application:

```bash
npm run build
npm run start
```

## Service Worker

The service worker is automatically generated by next-pwa and handles caching and offline functionality.

## Install Instructions

Users can install the app by:
1. Visiting the website in Chrome, Edge, or other Chromium-based browsers
2. Clicking the install button in the address bar
3. Or using the "Add to Home Screen" option

## Browser Support

PWA features are supported in:
- Chrome/Edge (Android & Desktop)
- Firefox (Android & Desktop)
- Safari (iOS)
- Samsung Internet
- Other modern browsers
